## 🔧 **修复内容**

### 问题原因
之前的 API 代码只在 `checkpoints/` 根目录查找 `.pth` 文件，而实际模型保存在子目录中：
```
checkpoints/
  └── all_days-5.msgs-...cell-gru/
      └── model.pth  ← 在这里
```

### 解决方案
修改了 `api/predictor_enhanced.py` 的 `_try_load_model` 方法：

1. **支持子目录搜索**：递归查找所有 `model.pth` 文件
2. **支持嵌套格式**：处理 `{'model_state_dict': ...}` 格式的检查点
3. **容错加载**：使用 `strict=False` 允许部分参数不匹配

### 关键代码改进
```python
# 修改前 (只查找根目录)
ckpt_files = [f for f in os.listdir(ckpt_dir) if f.endswith('.pth')]

# 修改后 (支持子目录)
for item in os.listdir('checkpoints'):
    if os.isdir(os.path.join('checkpoints', item)):
        model_file = os.path.join('checkpoints', item, 'model.pth')
        if os.path.exists(model_file):
            checkpoint_paths.append(model_file)
```
